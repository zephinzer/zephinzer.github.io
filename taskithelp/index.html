Searching for Items

Getting lost in the massive amount of work you have managed to get done with the assistance of your friendly TaskIt? No worries! We have a host of search facilities for you to effectively identify tasks you've created using TaskIt.

Current tasks

Format: search [search term]
Example: search lunch

Completed tasks

Format: search -a [search term]
Example: search business meeting

Dated tasks

Format: search -d [date]
Example: search 10 april 2014

Adding items

We have three types of items which you can add with the following commands:

Todo tasks (no time context)

Format: add [task name]
Example: add Call grandma

Deadlines tasks (ends at a given date & time)

Format: add [task name] by [date]
Example: add Business plan submission by today 4pm

Events tasks (lasts for a specified duration)

Format: add [task name] from [date1] to [date2]
Example: add Lunch with Boss from today 1pm to 3pm

Updating items

This is where it starts to get tricky but we promise it will pay off as you become increasingly familiar with the intuitive nature of TaskIt.

Updating requires the use of the item index as seen circled (in red) in the snapshot below:



Todo tasks

Updating of a Todo task is as simple as:
Format: update [index]; [new title here]
Example: update 5; A fresh new todo

Deadline tasks

Updating of a Deadline task is as simple as having another field:
Format: update [index]; [new title]; [new date]
Example 1: update 2; A wild Deadline appeared!; today 3pm;
Example 2: update 2; It was not very effective!; ;
Example 3: update 2; ; today 4pm;

For the above examples, Example 1 updates both the task title as well as the date & time, Example 2 updates only the task title, while Example 3 updates the date & time alone.

Events tasks

Updating of an Events task requires one last semi-colon:
Format: update [index]; [new title]; [new start date]; [new end date];
Example 1: update 2; A wild Event appeared!; today 3pm; tomorrow 4pm;
Example 2: update 2; It was not very effective!; ; ;
Example 3: update 2; ; today 4pm; ;
Example 4: update 2; ; ; tomorrow 3pm;

For the above examples, Example 1 updates both the task title as well as the date & times, Example 2 updates only the task title, while Example 3 updates the starting date & time alone and Example 4 updates the ending date & time alone.

Archiving items

Archiving an item from the TaskIt console requires the use of the item index as seen below within the red circle.


Archiving items is as simple as:
Format: archive [index]
Example: archive 3

In addition, we also have several variants of this command depending on your preference. Archive is also known as complete and even done. Both will work in a similar fashion to archive.
